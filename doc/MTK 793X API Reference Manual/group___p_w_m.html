Overview

This section introduces the Pulse-Width Modulation(PWM) APIs including terms and acronyms, supported features, software architecture, details on how to use this driver, PWM function groups, enums, structures and functions.

Terms and acronyms

Terms Details
PWM Pulse-Width Modulation. PWM is a modulation technique used to encode a message into a pulsing signal. For more information, please refer to PWM in Wikipedia .

Supported features

  • Support polling mode.
    The PWM generates the fixed waveform at the specified frequency and duty. The application can query current frequency and duty of the polling mode. The PWM does not support interrupt mode.

Software architecture of PWM

  1. PWM polling mode architecture.
    Call hal_pinmux_set_function() please refer to hal_pinmux_define.h. Not need to configure the pinmux, if EPT is used.
    Call hal_pwm_init() function to initialize the PWM source clock.
    Call hal_pwm_set_frequency() function to set the PWM frequency.
    Call hal_pwm_set_duty_cycle() function to set the PWM duty cycle. The duty cycle is calculated as the product of application's duty ratio.
    Call hal_pwm_start() function to trigger PWM execution.
    Call hal_pwm_get_frequency() and hal_pwm_get_duty_cycle() functions if the current frequency and duty cycle need to be retrieved.
    Call hal_pwm_get_running_status() function to get the PWM status, either busy or idle.
    Call hal_pwm_stop() function to stop the PWM execution.
    Polling mode architecture is similar to the polling mode architecture in HAL overview. See HAL Driver Model for polling mode architecture.

How to use this driver

  • Use PWM with polling mode.
    How to config PWM frequency and duty cycle at fixed souce clock. Assuming the crystal(XTAL) oscillator runs at 26MHz. The applicaton needs a waveform at 2kHz frequency and 50% duty ratio. The limits of frequency and duty cycle are as shown below.
    1. The maxinum supported frequency is the half of source clock frequency. When using the 26MHz Xtal as clock source, the maxinum supported frequency is 13MHz.
    2. PWM duty_cycle, the maxinum duty_cycle is 1000, resolution is 0.1, ex: duty_ratio is 50(represent duty cycle 50%), duty_cycle = (50 / 0.1) = 500.
    3. total_count = source_clock/frequency, and the value of total_count should greate than 2. It is optional and can be used to assist judgment.
    4. The frequency and duty cycle should meet the following formula.
      T(second) PWM period
      F (Hz) PWM frequency = 1/T
      t (second) Tick clk period
      f (Hz) Tick clk frequency = 1/t
      D (%) Duty cycle
      X (unit t) Value of configurable register pwm_on_time[15:0], maxinum value is 0xFFFF, in unit t
      Y (unit t) Value of configurable register pwm_off_time[15:0], maxinum value is 0xFFFF, in unit t
      Res (step) PWM resolution of duty cycle on certain F, f
      (X + Y) t = T
      (X + Y) = T/t = f/F =Res
      D = X / (X + Y)
      X = D * (X+Y) = D * f/F
      Y = f/F - X = f/F - D * f/F = ((1-D) * f)/F
      Error case: F = 200 Hz ,f = 26MHz, duty_cycle = 255, D = (duty_cycle / 1000) = ((duty_ratio / 0.1) / 1000)
      X = D * (X+Y) = D * f/F = 0x817E
      Y = f/F - X = f/F - D * f/F = ((1 - D) * f)/F = 0x17A52, exceed maxinum value 0xFFFF, will return error
      Correct case: F = 200 Hz ,f = 26MHz, duty_cycle = 500, D = duty_cycle / 1000 = ((duty_ratio / 0.1) / 1000)
      X = D * (X+Y) = D * f/F = 0xFDE8
      Y = f/F - X = f/F - D * f/F = ((1 - D * )f)/F = 0xFDE8, no exceed maxinum value 0xFFFF
  • Step1: Call hal_pinmux_set_function() to set the pwm pinmux, please refer to hal_pinmux_define.h. Not need to configure the pinmux, if EPT is used.
  • Step2: Call hal_pwm_init() to initialize the PWM source clock.
  • Step3: Call hal_pwm_set_frequency() to set frequency.
  • Step4: Call hal_pwm_set_duty_cycle() to set duty cycle according to application's duty ratio.
  • Step5: Call hal_pwm_start() to trigger PWM at a specified frequency and duty count values.
  • Step6: Call hal_pwm_get_frequency() to get the current frenquency.
  • Step7: Call hal_pwm_get_duty_cycle() to get the current duty_cycle.
  • Step8: Call hal_pwm_stop()to stop the PWM.
  • Step9: Call hal_pwm_deinit() to deinitialize the PWM.
  • Sample code:
    uint32_t duty_ratio = 50; //The range from 0 to 100, duty_cycle = duty_ratio * 10.
    hal_pwm_source_clock_t source_clock = HAL_PWM_CLOCK_26MHZ;
    uint32_t frequency = 2000;
    hal_gpio_init(HAL_GPIO_50);
    //function index = MT7933_PIN_50_FUNC_PWM_5; for more information about pinmux please refernence hal_pinmux_define.h
    hal_pinmux_set_function(HAL_GPIO_50, function_index);//Set the GPIO pinmux.
    //Initialize the PWM source clock.
    if(HAL_PWM_STATUS_OK != hal_pwm_init(source_clock)) {
    //error handle
    }
    //Sets frequency and gets total count of the PWM hardware at the specific frequency.
    if(HAL_PWM_STATUS_OK != hal_pwm_set_frequency(HAL_PWM_0, frequency, &total_count)) {
    //error handle
    }
    duty_cycle = duty_ratio * 10; //duty_cycle is calcauted as a product of application's duty_ratio.
    //Enable PWM to start the timer.
    if(HAL_PWM_STATUS_OK != hal_pwm_set_duty_cycle(HAL_PWM_0, duty_cycle)) {
    //error handle
    }
    hal_pwm_start(HAL_PWM_0); //Trigger PWM execution, start and stop API should appear in pairs.
    //...
    hal_pwm_get_frequency(HAL_PWM_0, &frequency);
    hal_pwm_get_duty_cycle(HAL_PWM_0, &duty_cycle);
    //...
    hal_pwm_stop(HAL_PWM_0); //Stop the PWM, due to suspend lock, start and stop API should appear in pairs.
    hal_pwm_deinit(); //Deinitialize the PWM.

hal_pwm_status_t hal_pwm_get_duty_cycle ( hal_pwm_channel_t  pwm_channel,
uint32_t *  duty_cycle 
)

This function gets the current duty cycle of the PWM.

Parameters
[in]pwm_channelis the PWM channel number. For more details about the parameter, please refer to hal_pwm_channel_t.
[out]*duty_cycleis PWM hardware duty cycle, which is calculated as a product of application's duty ratio and hardware 's total count. Due to the accuracy of HW, there is a deviation between the actual value and the preset value. For example: duty_cycle = 0xFF = 255, *duty_cycle will be about 0xFE.
Returns
To indicate whether this function call is successful or not. If the return value is HAL_PWM_STATUS_OK, the operation completed successfully. If the return value is HAL_PWM_STATUS_INVALID_PARAMETER, a wrong parameter is given. The parameter needs to be verified.
Example

hal_pwm_status_t hal_pwm_init ( hal_pwm_source_clock_t  source_clock)

This function initializes the PWM hardware source clock.

Parameters
[in]source_clockis the PWM source clock, it should be noted that the current driver only supports the clock of HAL_PWM_CLOCK_26MHZ. For more details about the parameter, please refer to hal_pwm_source_clock_t.
Returns
To indicate whether this function call is successful or not. If the return value is HAL_PWM_STATUS_OK, the operation completed successfully. If the return value is HAL_PWM_STATUS_INVALID_PARAMETER, a wrong parameter is given. The parameter needs to be verified.
Example
For a sample code, please refer to How to use this driver.
Note
There are some limitations about the minimum frequency that we can set if we choose the related source clock list in hal_pwm_source_clock_t, and the caculation formula is as follows:
The minimum frequency = ( the chosen source clock) / (0XFFFF * 2)
For example:
If we choose the clock of HAL_PWM_CLOCK_26MHZ, the minimum frequency that we can set is:HAL_PWM_CLOCK_26MHZ /( 0xFFFF * 2) = 199 Hz;
Warning
For this chip, all the PWM channel is using the same clock once this function has been called. It should be noted that the current driver only supports the clock of HAL_PWM_CLOCK_26MHZ.
