Overview
  • Project Demonstration
    • This project demonstrates the Wi-Fi/BT connectivity and Audio Service with low power feature
  • Application features
    • Act as a Wi-Fi station to connect to a Wi-Fi network.
    • Act as a Wi-Fi AP to accept connection to the MT7933 HDK using a handheld device or a laptop computer.
    • Ping out or into the device.
    • CM33 for Wi-Fi/BT Command Line Interface (CLI) via CM33_UART(baudrate: 921600) is supported in this project
    • DSP for Audio Command Line Interface (CLI) via DSP_UART(baudrate: 115200) is supported in this project.
Hardware and software environment
  • Supported platform
    • MediaTek MT7933 HDK.
  • HDK switches and pin configuration
    • The FTDI debug board can transfer USB interface to UART interface. Using this debug board can debug through UART, transmit and receive a signal form PC.
    • Two AMIC
    • J10 and J11 provide the pin for Speaker
    • Wi-Fi and BT function MT7933 main board reserve a Wi-Fi SMA connector and a BT SMA connector. Please connect external antenna to transmit and receive RF signals
    • There are some buttons on the board:
      • SW1 - reset.
      • SW2 - Vol+
      • SW3 - Vol-
      • SW4 - RTC_EINT
      • SW6 - DIP Switch : Switch board to Normal mode or Download mode.
    • USB slot
      • CON8 - CON8 is a USB 5V power for MT7933 main board, or you can use external 5V power at J1.
      • CON6 - CON6 is a USB function com port which do not supply 5V power to main board.
  • Environment configuration
    • A serial tool is required, such as hyper terminal or TeraTerm for UART logging.
    • COM port settings. baudrate: 921600, data bits: 8, stop bit: 1, parity: none and flow control: off.
  • See more information in MT7933_RFB_Users_Guide.pdf to download code.

Run the examples
  • Connect the FTDI board to a PC with a USB cable.
  • Build the example project and download the binary file to the MT793x HDK. See more information in MT793x_Flash_Burning_Tool_v2.77_ext.docx to Download code.
  • Reboot the HDK, the console will show "$" message to indicate the HDK is booting up.
  • Use '?' and enter to query the available command line options. Note that the command line options are still under development and subject to change without notice.
  • Below are 7 examples to demonstrate the Wi-Fi station, Wi-Fi access point modes of the 793x HDK, BT GATT mode, BT Mesh mode and Aduio example.
  • Example 1. Wi-Fi station mode (SEC mode).
    • Find your Wi-Fi access point settings: Before connecting to a Wi-Fi access point, the following information needs to be collected:
      1. The SSID of your Wi-Fi access point.
      2. The authentication mode of your Wi-Fi access point. In general, the authentication mode is WPA PSK or WPA2 PSK. To change the mode, please refer to Table of Authentication Modes for the list of supported authentication modes.
      3. The password of your Wi-Fi access point.
      4. The encryption mode of your Wi-Fi access point. In general, AES or TKIP is used. To change the mode, please refer to Table of Encyption Modes for the list of supported encryption modes.
    • Once the information is collected, use the following commands to configure the MT7933. The example below assumes either WPA PSK or WPA2 PSK is used for authentication, TKIP or AES for encryption, 'MTK_SOFT_AP' for the SSID, and the password of the WPA or WPA2 is '12345678' 
      wifi init
      wifi config set ssid 0 MTK_SOFT_AP
      wifi config set sec 0 9 8
      wifi config set psk 0 12345678
      wifi config set reload
    • After calling reload CLI by CM33_UART, if everything1` is correct, similar messages will be shown in the console to notify your HDK has received an IP address.
      DHCP got IP:192.168.1.100
    • ping: It assumes IP 19.168.1.100 is received. If the IP address is fetched and the network is operating, the MT7933 can ping other devices on the network with the following command in the console.
      ping 192.168.1.100
      The ping stops after sending three packets to 192.168.1.100.
      The ping usage is: ping <ip address> <times>

    • Find your Wi-Fi access point settings: Before connecting to a Wi-Fi access point, the following information needs to be collected:
      1. The SSID of your Wi-Fi access point.
    • Once the information is collected, use the following commands to configure the MT7933. The example below assumes, 'MTK_SOFT_AP' is used for the SSID. 
      wifi init
      wifi config set ssid 0 MTK_SOFT_AP

  • Example 3. Wi-Fi access point mode
    • Provide the Wi-Fi access point settings:
      1. SSID
      2. Authentication Mode
      3. Encryption Type
      4. Password
    • Once the information is collected, use the following commands to configure the MT7933. This example assumes WPA2 PSK is used for authentication, AES for encryption, 'MTK_SOFT_AP' for the SSID, and the password of the WPA2 is '87654321'
      wifi init
      wifi config set opmode 2
      wifi config set ssid 1 MTK_SOFT_AP
      wifi config set sec 1 7 6
      wifi config set psk 1 87654321
      wifi config set reload
    • After calling reload CLI by CM33_UART, use a handheld device or a laptop computer to connect to the access point 'MTK_SOFT_AP'. In the console, the IP address is shown on the handheld device or laptop.
  • Example 4. Connect to a Bluetooth LE device
    • Use the command "ble init" to initialize your Bluetooth module.
    • Input the command "ble gap start_scan 1 0140 0140 0 0 0" in the serial tool to scan for the nearby Bluetooth enabled devices.
      • 0, passive scan. 1, active scan.
      • 0140, scan interval (320 * 0.625 ms = 200ms). Two bytes long HEX value.
      • 0140, scan window (320 * 0.625 ms = 200ms). Two bytes long HEX value.
      • 0, own address type. 0, public address. 1, means random address.
      • 0, filter policy.
      • 0, filter duplicate option.
        //A similar log is written to the output:
        Find a device A,
        [I][APP] BT_GAP_LE_ADVERTISING_REPORT_IND Success
        [I][APP] ========================================
        [I][APP] Address: [RANDOM] 73-2c-d4-3f-d3-b2
        [I][APP] Event Type: ADV_IND
        [I][APP] Complete Name: MT793x Test Device
        [I][APP] ========================================
    • Input the command "ble gap stop_scan" to stop scan for the nearby Bluetooth enabled devices.
    • Connect the device with the command "ble gap connect 1 732cd43fd3b2".
      • 1, random address, based on the [RANDOM] tag. 0, public address based on the [PUBLIC] tag.
      • 732cd43fd3b2: remote device address. The output log indicates the connection is established with a remote device.
        [I][APP] BT_GAP_LE_CONNECT_IND Success
        [I][APP] connection handle=0x0200
        [I][APP] role=Master
        [I][APP] peer address:[RANDOM] 73-2c-d4-3f-d3-b2
  • Example 5. Start up as mesh provisionee device using CLI by CM33_UART
    • Switch bt config to mesh mode.
      ble boot mode mesh
      After execute command, system will auto reboot
    • Initialize your Bluetooth module.
      ble init
      After initialized, the mesh provisionee, will auto broadcast PB-ADV and PB-GATT.
    • Switch back to default GATT demo
      ble boot mode gatts
    • Delete the mesh data, use the command "mesh delete". It takes effect after rebooting again.
      mesh delete

        Audio_Uplink_Vol : 1 15
      3. Start VA program by CLI on CM33_UART
        aud_dbg va -v 1
        See log in DSP console
        [xxx.xxx][aud_t_va][time:xxxxxxxxxxxx]ww enter
